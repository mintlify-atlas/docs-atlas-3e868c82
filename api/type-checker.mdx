## Overview

The Type Checker is a constraint-based type inference engine that validates types throughout the Yoyo compilation pipeline. It uses a unification-based approach with domain constraints to solve type variables and ensure type safety.

## Core Components

### TypeChecker

The main type checker visitor that traverses the AST and generates type constraints.

```cpp
struct TypeChecker {
    std::optional<Type> target;
    IRGenerator* irgen;
    TypeCheckerState* state;
    bool in_mutable_ctx;
};
```

<ParamField path="target" type="std::optional<Type>">
  The expected type for the current expression being checked
</ParamField>

<ParamField path="irgen" type="IRGenerator*">
  Pointer to the IR generator for accessing type information
</ParamField>

<ParamField path="state" type="TypeCheckerState*">
  Current type checker state containing constraints and variables
</ParamField>

<ParamField path="in_mutable_ctx" type="bool">
  Whether the current context allows mutable operations
</ParamField>

## TypeCheckerState

Manages the state during type checking, including variables, constraints, and type unification.

```cpp
struct TypeCheckerState {
    std::vector<std::unordered_map<std::string, Type>> variables;
    std::vector<TypeCheckerConstraint> constraints;
    std::vector<TypeCheckerConstraint>* write_new_constraints_to;
    UnificationTable tbl;
    Type return_type;
};
```

<ParamField path="variables" type="std::vector<std::unordered_map<std::string, Type>>">
  Stack of variable scopes, each mapping variable names to their types
</ParamField>

<ParamField path="constraints" type="std::vector<TypeCheckerConstraint>">
  List of type constraints to be solved
</ParamField>

<ParamField path="tbl" type="UnificationTable">
  Union-find structure for type variable unification
</ParamField>

<ParamField path="return_type" type="Type">
  The return type of the function being checked
</ParamField>

## Domain

Represents the set of possible concrete types a type variable can be.

```cpp
class Domain {
    bool is_infinite = true;
public:
    struct Group {
        std::vector<Type> types;
        void add_type(Type&&);
    };
    
    Group concrete_types;
};
```

### Methods

```cpp
[[nodiscard]] std::optional<Error> add_and_intersect(Group&&, TypeCheckerState*);
[[nodiscard]] std::optional<Error> constrain_to_store(uint64_t);
[[nodiscard]] std::optional<Error> constrain_to_store(int64_t);
[[nodiscard]] std::optional<Error> constrain_to_store(double);
[[nodiscard]] std::optional<Error> merge_intersect(Domain&&, TypeCheckerState*);
[[nodiscard]] std::optional<Error> equal_constrain(Type other);
bool is_solved();
Type get_solution();
```

<ResponseField name="add_and_intersect">
  Adds a group of possible types and intersects with the current domain
</ResponseField>

<ResponseField name="constrain_to_store">
  Constrains the domain to types that can store the given value
</ResponseField>

<ResponseField name="is_solved" type="bool">
  Returns true if the domain has been reduced to a single type
</ResponseField>

<ResponseField name="get_solution" type="Type">
  Returns the solved type if the domain is solved
</ResponseField>

## Type Constraints

The type checker uses a variety of constraints to express type requirements:

<Expandable title="Numeric Constraints">

### IsIntegerConstraint
```cpp
struct IsIntegerConstraint {
    Type type;
    ASTNode* expr;
};
```
Ensures a type is an integer (i8-i64/u8-u64).

### CanStoreIntegerConstraint
```cpp
struct CanStoreIntegerConstraint {
    Type type;
    std::variant<uint64_t, int64_t> value;
    ASTNode* expr;
};
```
Ensures a type can store a specific integer value.

### IsFloatConstraint
```cpp
struct IsFloatConstraint {
    Type type;
    ASTNode* expr;
};
```
Ensures a type is a floating point value (f32/f64).

### CanStoreRealConstraint
```cpp
struct CanStoreRealConstraint {
    Type type;
    double value;
    ASTNode* expr;
};
```
Ensures a type can store a specific floating point value.

</Expandable>

<Expandable title="Operator Constraints">

### BinaryOperableConstraint
```cpp
struct BinaryOperableConstraint {
    Type left;
    Type right;
    Type result;
    TokenType op;
    std::unordered_map<OperatorOverload*, std::unordered_map<std::string, Type>> substitution_cache;
    ASTNode* expr;
};
```
Ensures a binary operation `left op right` is defined and returns `result`.

### HasUnaryMinusConstraint
```cpp
struct HasUnaryMinusConstraint {
    Type type;
    Type ret;
    ASTNode* expr;
};
```
Ensures a type supports unary minus.

### HasUnaryNotConstraint
```cpp
struct HasUnaryNotConstraint {
    Type type;
    Type ret;
    ASTNode* expr;
};
```
Ensures a type supports unary not.

### IndexOperableConstraint
```cpp
struct IndexOperableConstraint {
    Type left;
    Type right;
    Type result;
    bool is_mutable;
    ASTNode* expr;
};
```
Ensures subscript operation `left[right]` is valid.

</Expandable>

<Expandable title="Reference Constraints">

### IsReferenceToConstraint
```cpp
struct IsReferenceToConstraint {
    Type type;
    Type other;
    ASTNode* expr;
};
```
Ensures `type` is `&other` or `&mut other`.

### IsNotReferenceConstraint
```cpp
struct IsNotReferenceConstraint {
    Type type;
    ASTNode* expr;
};
```
Ensures a type is not a reference.

### OwningConstraint
```cpp
struct OwningConstraint {
    Type type;
    ASTNode* expr;
};
```
Ensures a type is owning.

### NonOwningConstraint
```cpp
struct NonOwningConstraint {
    Type type;
    ASTNode* expr;
};
```
Ensures a type is non-owning (a reference).

### BorrowResultConstraint
```cpp
struct BorrowResultConstraint {
    Type subject;
    Type result;
    ASTNode* expr;
    std::unordered_map<OperatorOverload*, std::unordered_map<std::string, Type>> substitution_cache;
};
```
Ensures borrowing `subject` results in `result`.

</Expandable>

<Expandable title="Function Constraints">

### IsInvocableConstraint
```cpp
struct IsInvocableConstraint {
    Type type;
    ASTNode* expr;
};
```
Ensures a type can be called (function or lambda).

### ValidAsFunctionArgConstraint
```cpp
struct ValidAsFunctionArgConstraint {
    Type type;
    Type function;
    uint32_t arg_no;
    ASTNode* expr;
};
```
Ensures a type is valid as a specific function argument.

### IsReturnOfConstraint
```cpp
struct IsReturnOfConstraint {
    Type type;
    Type function;
    ASTNode* expr;
};
```
Ensures a type is the return type of a function.

</Expandable>

<Expandable title="Structural Constraints">

### EqualConstraint

Ensures two types are equal.

### HasFieldConstraint
```cpp

    Type subject;
```

Ensures a type has a specific field.

### AllFieldsConstraint
```cpp

    std::vector<std::string> fields;

    ASTNode* expr;

```
Ensures a type has all specified fields.

### ImplInterfaceConstraint

```cpp

```

## Usage in Pipeline

The type checker operates after parsing and before IR generation:

1. **Parse** - AST is constructed
2. **Type Check** - Constraints are generated and solved
3. **Borrow Check** - Ownership and lifetimes are verified
4. **IR Generation** - Typed AST is lowered to IR

The type checker ensures that all type annotations are correct, all operations are type-safe, and all generic parameters are properly instantiated before code generation begins.
