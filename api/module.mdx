## Overview

The `ModuleBase` class is the foundation for Yoyo's module system. It manages all declarations within a module including functions, classes, types, constants, and nested modules. Each module maintains symbol tables for different kinds of declarations and provides methods for symbol lookup and resolution.

## Class Definition

```cpp
namespace Yoyo {
    class ModuleBase
    {
    public:
        struct FunctionDetails {
            std::string name;
            FunctionSignature sig;
            std::vector<Attribute> attributes;
            bool is_private() const;
        };
        
        virtual ~ModuleBase() = default;
        
        // Public members
        ModuleOverloadDetails overloads;
        Engine* engine;
        std::unordered_map<std::string, ModuleBase*> modules;
        std::string module_hash;
        
        // Declaration storage
        std::unordered_map<std::string, std::vector<FunctionDetails>> functions;
        std::unordered_map<std::string, std::vector<std::pair<std::string, std::unique_ptr<ClassDeclaration>>>> classes;
        std::unordered_map<std::string, std::vector<std::unique_ptr<UnionDeclaration>>> unions;
        std::unordered_map<std::string, std::vector<std::unique_ptr<GenericFunctionDeclaration>>> generic_fns;
        std::unordered_map<std::string, std::vector<std::unique_ptr<InterfaceDeclaration>>> interfaces;
        std::unordered_map<std::string, std::vector<std::unique_ptr<GenericAliasDeclaration>>> generic_aliases;
        std::unordered_map<std::string, std::vector<std::unique_ptr<GenericInterfaceDeclaration>>> generic_interfaces;
        std::unordered_map<std::string, std::unordered_map<std::string, Type>> aliases;
        std::unordered_map<std::string, std::vector<std::unique_ptr<GenericClassDeclaration>>> generic_classes;
        std::unordered_map<std::string, std::vector<std::unique_ptr<EnumDeclaration>>> enums;
        std::unordered_map<std::string, std::vector<std::tuple<Type, std::string, std::variant<Constant, ConstantDeclaration*>>>> constants;
        std::unordered_map<std::string, std::vector<std::unique_ptr<UnionDeclaration>>> generic_unions;
        std::unordered_map<std::string, std::vector<std::unique_ptr<MacroDeclaration>>> macros;
        
        // Symbol lookup methods
        std::pair<std::string, FunctionDetails*> findFunction(const std::string& block, const std::string& name);
        std::pair<std::string, GenericFunctionDeclaration*> findGenericFn(const std::string& block, const std::string& name);
        Type* findAlias(const std::string& block, const std::string& name);
        std::pair<std::string, GenericAliasDeclaration*> findGenericAlias(const std::string& block, const std::string& name);
        std::pair<std::string, GenericClassDeclaration*> findGenericClass(const std::string& block, const std::string& name);
        std::pair<std::string, InterfaceDeclaration*> findInterface(const std::string& block, const std::string& name);
        std::pair<std::string, GenericInterfaceDeclaration*> findGenericInterface(const std::string& block, const std::string& name);
        std::pair<std::string, EnumDeclaration*> findEnum(const std::string& block, const std::string& name);
        MacroDeclaration* findMacro(const std::string& block, const std::string& name);
        std::pair<std::string, UnionDeclaration*> findUnion(const std::string& block, const std::string& name);
        std::pair<std::string, std::pair<std::string, std::unique_ptr<ClassDeclaration>>*> findClass(const std::string& block, const std::string& name);
        std::pair<std::string, std::tuple<Type, std::string, std::variant<Constant, ConstantDeclaration*>>*> findConst(const std::string& block, const std::string& name);
        std::optional<std::string> hashOf(const std::string& base_block, const std::string& name);
    };
}
```

## FunctionDetails Structure

Contains metadata about a function declaration.

```cpp
struct FunctionDetails {
    std::string name;
    FunctionSignature sig;
    std::vector<Attribute> attributes;
    bool is_private() const;
};
```

### is_private()

Determines if a function is private (lacks a `public` attribute).

```cpp
bool is_private() const;
```

<ResponseField name="return" type="bool">
  true if the function is private, false if it has a public attribute
</ResponseField>

## Declaration Storage

The module stores different kinds of declarations in separate maps. Each map is keyed by block hash (namespace identifier) and contains vectors of declarations to support overloading.

### functions

Stores non-generic function declarations.

```cpp
std::unordered_map<std::string, std::vector<FunctionDetails>> functions;
```

**Key:** Block hash
**Value:** Vector of function details (supports overloading)

### generic_fns

Stores generic (template) function declarations.

```cpp
std::unordered_map<std::string, std::vector<std::unique_ptr<GenericFunctionDeclaration>>> generic_fns;
```

### classes

Stores non-generic class declarations.

```cpp
std::unordered_map<std::string, std::vector<std::pair<std::string, std::unique_ptr<ClassDeclaration>>>> classes;
```

Each class is stored as a pair containing its hash and declaration.

### generic_classes

Stores generic (template) class declarations.

```cpp
std::unordered_map<std::string, std::vector<std::unique_ptr<GenericClassDeclaration>>> generic_classes;
```

### interfaces

Stores non-generic interface declarations.

```cpp
std::unordered_map<std::string, std::vector<std::unique_ptr<InterfaceDeclaration>>> interfaces;
```

### generic_interfaces

Stores generic (template) interface declarations.

```cpp
std::unordered_map<std::string, std::vector<std::unique_ptr<GenericInterfaceDeclaration>>> generic_interfaces;
```

### unions

Stores non-generic union type declarations.

```cpp
std::unordered_map<std::string, std::vector<std::unique_ptr<UnionDeclaration>>> unions;
```

### generic_unions

Stores generic (template) union type declarations.

```cpp
std::unordered_map<std::string, std::vector<std::unique_ptr<UnionDeclaration>>> generic_unions;
```

### enums

Stores enum declarations.

```cpp
std::unordered_map<std::string, std::vector<std::unique_ptr<EnumDeclaration>>> enums;
```

### aliases

Stores type alias declarations (non-generic).

```cpp
std::unordered_map<std::string, std::unordered_map<std::string, Type>> aliases;
```

**Outer key:** Block hash
**Inner key:** Alias name
**Value:** The type being aliased

### generic_aliases

Stores generic (template) type alias declarations.

```cpp
std::unordered_map<std::string, std::vector<std::unique_ptr<GenericAliasDeclaration>>> generic_aliases;
```

### constants

Stores constant declarations.

```cpp
std::unordered_map<std::string, std::vector<std::tuple<Type, std::string, std::variant<Constant, ConstantDeclaration*>>>> constants;
```

Each constant is stored as a tuple containing:
1. The constant's type
2. The constant's name
3. Either a computed Constant value or pointer to the declaration

### macros

Stores macro declarations.

```cpp
std::unordered_map<std::string, std::vector<std::unique_ptr<MacroDeclaration>>> macros;
```

## Symbol Lookup Methods

All lookup methods search within a specific block (namespace) and support hierarchical lookup through parent blocks.


## Usage Example

```cpp
// Create a module
auto module = std::make_unique<LLVMModule>();
module->engine = engine.get();
module->module_hash = "main_module_hash";

// Add a function to the module
FunctionDetails func_details;
func_details.name = "add";
func_details.sig = /* ... */;
func_details.attributes.push_back({"public"});

module->functions["main_block"].push_back(std::move(func_details));

// Look up the function
auto [block_hash, func_ptr] = module->findFunction("main_block", "add");
if (func_ptr) {
    std::cout << "Found function: " << func_ptr->name << std::endl;
    std::cout << "Is private: " << func_ptr->is_private() << std::endl;
}

// Add the module to the engine
engine->modules["main"] = std::move(module);
```

## Block Hashes

Block hashes are used throughout the module system to identify namespaces and scopes. They enable:
- Symbol disambiguation in nested scopes
- Privacy control (private vs public symbols)
- Hierarchical symbol lookup
- Module composition

The hash system allows symbols with the same name to coexist in different blocks without conflicts.
