## Overview

The `Parser` class is responsible for converting Yoyo source code into an Abstract Syntax Tree (AST). It uses a Pratt parser design with pluggable prefix and infix parselets for extensibility.

## Class Definition

```cpp
namespace Yoyo {
    class YOYO_API Parser
    {
    public:
        explicit Parser(std::string& source);
        
        // Main parsing methods
        std::unique_ptr<ASTNode> parseExpressionOrDeclaration();
        std::unique_ptr<Expression> parseExpression(uint32_t precedence);
        std::vector<std::unique_ptr<Statement>> parseProgram();
        std::optional<Type> parseType(uint32_t precedence);
        
        // Statement parsing
        std::unique_ptr<Statement> parseStatement();
        std::unique_ptr<Statement> parseDeclaration();
        std::unique_ptr<Statement> parseExpressionStatement();
        
        // Declaration parsing
        std::unique_ptr<Statement> parseVariableDeclaration(Token identifier);
        std::unique_ptr<Statement> parseEnumDeclaration(Token identifier);
        std::unique_ptr<Statement> parseConstDeclaration(Token identifier);
        std::unique_ptr<Statement> parseClassDeclaration(Token identifier, bool);
        std::unique_ptr<Statement> parseAliasDeclaration(Token identifier);
        std::unique_ptr<Statement> parseInterfaceDeclaration(Token identifier);
        std::unique_ptr<Statement> parseUnionDeclaration(Token identifier);
        std::unique_ptr<Statement> parseMacroDeclaration(Token identifier);
        std::unique_ptr<Statement> parseUsingDeclaration(Token);
        
        // Control flow parsing
        std::unique_ptr<Statement> parseReturnStatement(Token);
        std::unique_ptr<Statement> parseForStatement(Token);
        std::unique_ptr<Statement> parseWhileStatement(Token);
        std::unique_ptr<Statement> parseWithStatement(Token);
        std::unique_ptr<Statement> parseBreakContinueStatement(Token tk);
        
        // Expression parsing
        std::unique_ptr<Expression> parseConditionalExtraction(Token);
        std::unordered_map<std::string, std::unique_ptr<Expression>> parseObjectLiteral();
        
        // Signature and clause parsing
        std::optional<FunctionSignature> parseFunctionSignature();
        std::optional<GenericClause> parseGenericClause();
        std::vector<Attribute> parseAttributeList();
        UsingStatement::ContentTy parseUsingContent(uint32_t);
        
        // Token management
        std::optional<Token> Peek();
        std::optional<Token> Get();
        std::optional<std::pair<Token, SourceLocation>> PeekWithEndLocation();
        std::optional<std::pair<Token, SourceLocation>> GetWithEndLocation();
        void pushToken(Token t, SourceLocation loc);
        bool discard(TokenType t);
        
        // Error handling
        void error(std::string message, std::optional<Token> tk);
        void synchronizeTo(std::span<const TokenType> t);
        bool failed() const;
        
        // Precedence
        uint32_t GetNextTypePrecedence();
        
        // Utility methods
        bool canOmitSemiColon(Expression*);
        const Scanner& getScanner() const;
        const std::string& getSource() const;
        
        // Public members
        ASTNode* parent = nullptr;
        SourceLocation discardLocation;
    };
}
```

## Constructor

### Parser()

Constructs a parser for the given source code.

```cpp
explicit Parser(std::string& source);
```

<ParamField path="source" type="std::string&">
  Reference to the source code string to parse. The reference must remain valid for the lifetime of the Parser.
</ParamField>

## Core Parsing Methods

### parseProgram()

Parses the entire source file into a vector of top-level statements.

```cpp
std::vector<std::unique_ptr<Statement>> parseProgram();
```

<ResponseField name="return" type="std::vector<std::unique_ptr<Statement>>">
  Vector of parsed top-level statements (functions, classes, variables, etc.)
</ResponseField>

### parseExpression()

Parses an expression with the given precedence level using Pratt parsing.

```cpp
std::unique_ptr<Expression> parseExpression(uint32_t precedence);
```

<ParamField path="precedence" type="uint32_t">
  Minimum precedence level for parsing. Use 0 for lowest precedence.
</ParamField>

<ResponseField name="return" type="std::unique_ptr<Expression>">
  Parsed expression node
</ResponseField>

### parseExpressionOrDeclaration()

Parses either an expression or a declaration, automatically detecting which.


<ResponseField name="return" type="std::unique_ptr<ASTNode>">

### parseType()

<ParamField path="precedence" type="uint32_t">
  Precedence level for type parsing
</ParamField>

<ResponseField name="return" type="std::optional<Type>">

```cpp

```

<ResponseField name="return" type="std::optional<Token>">
  The next token, or std::nullopt if at end of file

## Usage Example

```cpp
std::string source = R"(
    fn main() {
        println("Hello, Yoyo!");
    }
)";

Parser parser(source);
auto program = parser.parseProgram();

if (parser.failed()) {
    std::cerr << "Parsing failed!" << std::endl;
    return 1;
}

// Process the AST
for (auto& stmt : program) {
    // Process each statement
}
```
