## Overview

The Borrow Checker ensures memory safety in Yoyo by tracking ownership, borrows, and lifetimes. It converts the typed AST into a Control Flow Graph (CFG) with domain constraints, then performs dataflow analysis to verify that all memory accesses are safe.

The borrow checker uses an SSA-based intermediate representation where values flow through basic blocks, and domain variables track the lifetimes of borrowed references.

## Domain Tracking

### Domain

Represents a lifetime/borrow region in the program.

```cpp
struct Domain {
    std::string name;
    std::string to_string() const { return name; }
    bool is_var() const { return name.starts_with("'?"); }
    bool is_null() const { return name == "null"; }
    bool operator==(const Domain& other) const { return name == other.name; }
};
```

Domains are used to track the lifetimes of references. Domain variables (starting with `'?`) are unified during analysis.

## Analysis Algorithms

### Points-To Analysis

The borrow checker uses inclusion-based points-to analysis to determine which memory locations each reference might point to.

```cpp
struct PointsToGraph {
    std::unordered_map<std::string, std::unordered_set<std::string>> pointee_pairs;
    
    bool add_edge(std::string domain, std::string pointee);
    std::unordered_set<std::string>& get_pointees_of(std::string domain);
    std::string to_graphviz();
};
```

### Domain Validity Analysis

Tracks which domains are "alive" at each program point using dataflow analysis:

```cpp
struct KillReason {
    enum InstructionSource { Assign, Drop };
    InstructionSource source;
    std::string affected_value;
    std::string bad_pointee;
    ASTNode* killing_instruction;
    std::optional<std::string> lvalue_source;
};
```

When a value is dropped or reassigned, all domains it points to are "killed". The analysis ensures no reference is used after its domain has been killed.

## Usage in Pipeline

The borrow checker runs after type checking:

1. **Type Check** - Resolves all types
2. **Emit CFG** - Convert AST to borrow checker IR
3. **Insert Domains** - Annotate with lifetime variables
4. **SSA Transform** - Convert to SSA form
5. **Points-To Analysis** - Determine aliasing relationships
6. **Validity Analysis** - Ensure no use-after-free
7. **IR Generation** - Proceed to code generation

This ensures memory safety without garbage collection or manual memory management.
