# Welcome to Yoyo

Yoyo is a statically typed, memory-safe scripting language designed to be embedded in C++20 applications. With JIT compilation and a modern syntax inspired by cppfront, Yoyo brings safety and performance to embedded scripting.

<Note>
  This implementation is currently in early development and is not yet production-ready.
</Note>

## Quick Example

Here's a simple Yoyo program demonstrating the language's syntax:

```yoyo
// Define a struct with methods
Vector2: struct = {
    x: u32,
    y: u32,
    
    add: fn (&this, other: &Vector2) -> Vector2 = {
        return Vector2{ .x = this.x + other.x, .y = this.y + other.y };
    }
}

// Main function
main: fn = {
    a := Vector2{ .x = 10, .y = 20 };
    b := Vector2{ .x = 5, .y = 15 };
    result := a.add(&b);
    
    "Result: ${result.x}, ${result.y}".test::print();
}
```

## Design Philosophy

Yoyo's design focuses on three core principles:

- **Safety First**: Memory safety through borrow checking, preventing common bugs at compile time
- **Performance**: JIT compilation for near-native execution speeds
- **Developer Experience**: Modern syntax that's both expressive and easy to learn

## Use Cases

Yoyo is ideal for:

- **Game Scripting**: Embed safe, fast scripts in C++ game engines
- **Application Extensions**: Allow users to extend your C++ application with custom logic
- **Configuration**: Complex configuration that goes beyond static files
- **Hot Reloading**: Update application logic without recompilation
